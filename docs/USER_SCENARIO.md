## 2. 핵심 유저 시나리오

### 시나리오 A: 아침 업무 시작 (7분 → 30초)

**AS-IS (기존 방식)**
```
1. PC 켜기
2. 크롬 열기
3. Gmail 탭 열기 → 메일 확인 (2~3분)
4. Teams 탭 열기 → 어제 놓친 메시지 확인 (2~3분)
5. Notion 열기 → 오늘 할 일 파악 (1~2분)
6. Google Calendar 열기 → 오늘 일정 확인 (30초)
7. Figma 열기 → 파일 검색 (30초~1분)
총 소요 시간: 6~9분, 탭 전환: 5회
```

**TO-BE (MyDesk 사용)**
```
1. PC 켜기
2. MyDesk 열기 (브라우저 기본 탭으로 설정)
3. 커뮤니케이션 피드에서 메일+Teams 통합 확인 (30초)
4. 구글 캘린더 위젯으로 오늘 일정 확인 (10초)
5. 업무 시트에서 오늘 할 일 점검 (10초)
6. 피그마 퀵런처에서 작업 파일 클릭 (1초)
총 소요 시간: ~1분, 탭 전환: 0~1회
```

---

### 시나리오 B: AI 보조를 활용한 기획서 초안 작성

**상황**: 진료화면 리뉴얼 요구사항 정리 중, AI에게 UX 패턴 질문이 필요함

**AS-IS**
```
1. "AI한테 물어봐야지"
2. 새 탭 → chatgpt.com 접속 (로딩 3~5초)
3. 채팅창 클릭 → 질문 입력
4. 다시 Figma 탭으로 돌아옴 → "어디까지 했더라?"
→ 컨텍스트 복구 비용 발생
```

**TO-BE**
```
1. MyDesk 우측 AI 어시스턴트 허브 탭 클릭
2. Groq (Llama) 탭에서 바로 질문 입력
3. 답변 받으면서 왼쪽 Figma 퀵런처로 바로 이동
→ 화면 이탈 없음, 컨텍스트 유지
```

---

### 시나리오 C: 집중 작업 세션 (뽀모도로)

**상황**: 기획서 작성 집중이 필요한 오후 2시

**플로우**
```
1. MyDesk 우측 뽀모도로 타이머 → "시작" 클릭
2. 25분 카운트다운 시작
3. 데일리 노트에 이 세션에서 완료할 것 적어둠
4. 타이머 종료 → 세션 도트 1개 추가
5. 5분 휴식 → AI 조언 배너의 "다음 단계" 확인
6. 반복
→ 방해 요소 없이 집중 패턴 형성
```

---

### 시나리오 D: 피그마 파일 빠른 접근

**상황**: 오전에 4개 프로젝트를 번갈아 작업

**플로우**
```
1. 피그마 퀵런처 위젯에 등록된 4개 파일 카드 확인
2. "진료화면 리뉴얼 v3" 카드 클릭 → figma.com 새 탭
3. 수정 후 다시 MyDesk로 돌아옴
4. "EMR 대시보드 컴포넌트" 카드 클릭 → 즉시 이동
→ Figma 파일 탐색 시간: 10초 → 1초
```

---

### 시나리오 E: 미읽음 알림 빠른 처리

**상황**: 회의 중 Teams와 메일 알림이 쌓임

**플로우**
```
1. 회의 종료 후 MyDesk 커뮤니케이션 피드 확인
2. 통합 타임라인에서 Teams 2건 + 메일 5건 확인
3. 소스 색상 배지(파랑=메일, 시안=Teams)로 분류 파악
4. 중요도 높은 메시지 우선 처리
→ 별도 앱 전환 없이 무엇이 왔는지 한눈에 파악
```

## 3. 태스크 플로우 다이어그램

```
[MyDesk 접속]
      │
      ├─ AI 조언 배너 확인 (오늘의 컨텍스트 파악)
      │
      ├─ 퀵 앱 바: Figma / Claude / Teams / Mail 바로가기
      │
      ├─ 좌측 (2/3)
      │   ├─ 피그마 퀵런처 → 작업 파일 1클릭 오픈
      │   ├─ AI 어시스턴트 → Groq 탭으로 즉시 질문
      │   ├─ 업무 시트 → 태스크 상태 업데이트
      │   └─ 구글 캘린더 → 오늘 일정 인라인 확인
      │
      └─ 우측 (1/3)
          ├─ 뽀모도로 타이머 → 집중 세션 시작
          ├─ 데일리 노트 → 체크리스트 + 메모
          ├─ 커뮤니케이션 피드 → 메일+Teams 통합
          └─ AI·기획 뉴스 → 최신 트렌드 빠른 확인
```

## 5. 엣지 케이스 및 예외 처리

| 상황 | 처리 방식 |
|------|----------|
| Groq API 호출 실패 | 친절한 에러 메시지 + 재시도 안내 |
| 데스크탑 앱 미설치 | 1.5초 대기 후 웹 URL로 fallback |
| 캘린더 미연결 | 이메일 입력 안내 화면 표시 |
| 피그마 파일 0개 | 빈 상태 가이드 UI + 추가 유도 |
| 뉴스 RSS 로딩 실패 | 에러 상태 + 재시도 버튼 표시 |
| localStorage 초과 | 최대 항목 제한 (피그마: 8개, 태스크: 제한 없음) |
