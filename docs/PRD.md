# MyDesk PRD (Product Requirements Document)

**버전**: 1.0.0
**작성일**: 2026-03-13
**작성자**: 기획자 (EMR 업계)
**상태**: MVP 개발 완료

---

## 1. 문제 정의 (Problem Statement)

### 1.1 배경

EMR(전자의무기록) 업계 기획자는 하루 평균 **7~10개 이상의 툴**을 넘나들며 업무를 수행한다.

| 도구 | 용도 |
|------|------|
| Figma | UI/UX 설계, 프로토타이핑 |
| Teams / Slack | 팀 커뮤니케이션 |
| Gmail / Outlook | 이메일 확인 |
| Notion / Confluence | 기획 문서 관리 |
| Excel / Google Sheets | 업무 추적 |
| Claude / ChatGPT / Gemini | AI 보조 |
| Google Calendar | 일정 관리 |

### 1.2 핵심 문제

> **"컨텍스트 스위칭 비용이 너무 크다"**

- 툴을 전환할 때마다 **집중 흐름(flow)이 끊김**
- 어떤 메시지를 읽었는지, 어떤 태스크가 남았는지 **파악에 시간 소요**
- 여러 AI 도구를 각각 열어야 해서 **AI 활용 마찰이 큼**
- 뽀모도로 같은 집중 관리 도구를 **별도로 실행해야 함**

### 1.3 영향

- 기획자 1인 기준, 하루 **40~60분이 툴 전환 및 컨텍스트 파악에 낭비**
- 중요 메시지·마감 놓침 → 품질 저하, 신뢰 손상
- AI를 활용하고 싶어도 진입 장벽으로 활용률 저조

## 2. 제품 목표 (Goals)

### 2.1 비전

> **"EMR 기획자를 위한 AI-Native 올인원 업무 허브"**
> 한 화면에서 모든 업무 맥락을 유지하며 일한다.

### 2.2 목표

| # | 목표 | 측정 방법 |
|---|------|----------|
| G1 | 툴 전환 횟수 50% 감소 | 일 평균 앱 전환 횟수 측정 |
| G2 | 업무 시작까지 걸리는 시간 단축 (< 30초) | 첫 액션까지 소요 시간 |
| G3 | AI 활용 빈도 증가 | 일 평균 AI 어시스턴트 사용 횟수 |
| G4 | 마감/메시지 누락 제로화 | 미읽은 알림 처리율 |

### 2.3 MVP 범위

이번 버전은 **혼자 쓰는 개인 대시보드 MVP**로, 실서버 연동보다 **동작하는 완성도**에 집중한다.

## 3. 사용자 정의 (Users)

### 주요 타겟

- **직군**: EMR 소프트웨어 회사 서비스 기획자
- **경력**: 2~8년차
- **특징**: 기술 이해도 높음, AI 툴 사용 경험 있음, 협업 툴 의존도 높음

### 유저 스토리

```
As a EMR 기획자,
I want to 하나의 화면에서 오늘의 태스크, AI 어시스턴트, 커뮤니케이션을 모두 확인하고 싶다
So that 툴 전환 없이 집중해서 일할 수 있다.
```

## 4. 기능 명세 (Feature Specification)

### 4.1 위젯 목록

#### REQ-01: 대시보드 레이아웃
- 2/3 + 1/3 그리드 레이아웃
- 위젯 표시/숨기기 토글
- 위젯 순서 변경 (위/아래)
- 레이아웃 프리셋 저장 (최대 3개)

#### REQ-02: 위젯 설정 모달
- 헤더 "설정" 버튼 → 모달 열기
- 각 위젯 표시/숨기기
- 위젯 순서 조정
- 프리셋: 기획 집중 모드 / 회의 많은 날 / 커스텀

#### REQ-03: 피그마 퀵런처
- Figma 파일 URL + 파일명 등록 (최대 8개)
- 색상 카드 썸네일 + 파일명 + 등록 시간 표시
- 카드 클릭 → 새 탭으로 Figma 오픈
- X 버튼 또는 우클릭 메뉴로 삭제
- localStorage 영속 저장

#### REQ-04: AI 어시스턴트 허브
- Groq(Llama 3.3), ChatGPT, Gemini 탭 전환
- Groq API 실시간 연동 (llama-3.3-70b-versatile)
- 대화 히스토리 스크롤
- ChatGPT / Gemini는 준비 중 표시
- 에러 시 친절한 메시지

#### REQ-05: 뽀모도로 타이머
- 25분 집중 / 5분 휴식 사이클
- 시작 / 일시정지 / 리셋
- 오늘 완료 세션 도트 표시 (최대 4개)
- 자동 전환

#### REQ-06: 업무 시트 뷰어
- Task / Status / Due / Priority 컬럼 테이블
- 상태 클릭 사이클: 대기 → 진행중 → 완료 → 보류
- 우선순위 색상 뱃지 (높음: 빨강, 보통: 노랑, 낮음: 회색)
- 태스크 추가 / 삭제
- localStorage 영속 저장

#### REQ-07: 데일리 노트
- 날짜별 체크리스트 + 자유 메모
- 완료 항목 아래로 정렬
- 오늘 날짜 자동 키 생성

#### REQ-08: 커뮤니케이션 피드
- 메일 + Teams 통합 피드 (목 데이터)
- 미읽음 카운트 뱃지
- 소스별 색상 구분 (메일: 파랑, Teams: 시안)

#### REQ-09: AI 조언 배너
- AI 조언 탭: 업무 컨텍스트 기반 조언 랜덤 로테이션
- 오늘의 운세 탭: Groq API로 생성, 하루 1회 캐싱
- 닫기 버튼

#### REQ-10: AI·기획 뉴스 피드
- Google News RSS 실시간 파싱 (서버사이드)
- AI기술 / UX기획 / 디자인툴 / 헬스IT 카테고리 자동 분류
- 30분 서버 캐시 + 클라이언트 캐시

#### REQ-11: Google 캘린더 뷰
- Google Calendar 공식 임베드 iframe
- 주간 / 월간 / 일정 뷰 전환
- 계정 이메일 입력으로 연결

#### REQ-12: 즐겨쓰는 앱 바로가기
- Figma, Claude, ChatGPT, Gemini, Teams, Excel, Mail
- 데스크탑 앱 프로토콜 우선 실행 (msteams://, figma:// 등)
- 앱 미설치 시 웹 fallback (1.5초 timeout)
- Teams, Mail 미읽음 뱃지 표시 (목 데이터)

### 4.2 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 성능 | 초기 로딩 3초 이내 |
| 저장 | localStorage 기반 (서버 DB 없음, MVP) |
| 보안 | API 키 서버사이드 전용, 클라이언트 미노출 |
| 접근성 | 키보드 네비게이션, aria 라벨 |
| 반응형 | 1280px 이상 데스크탑 최적화 |

## 7. 디자인 원칙

- **다크 글래스모피즘**: 밤하늘 배경 + 반투명 카드
- **컨텍스트 유지**: 모든 정보를 한 화면에
- **AI-Native**: AI 조언, 운세, 뉴스가 자연스럽게 통합
- **플랫 & 클린**: 그라디언트/그림자 최소화

## 8. 검증 계획 (Validation Plan)

### 8.1 핵심 가설

| # | 가설 | 검증 방법 |
|---|------|----------|
| H1 | 기획자는 하루 5회 이상 대시보드를 방문한다 | 세션 횟수 측정 |
| H2 | AI 어시스턴트를 통해 기획 질문 응답 시간이 50% 단축된다 | 기존 대비 작업 시간 비교 |
| H3 | 피그마 퀵런처 사용으로 Figma 접근 시간이 10초 → 1초로 단축된다 | 클릭 횟수 및 소요 시간 |
| H4 | 업무 시트를 매일 업데이트하는 사용자가 70% 이상이다 | localStorage 데이터 갱신 빈도 |
| H5 | 컨텍스트 스위칭 횟수가 40% 이상 감소한다 | 브라우저 탭 전환 수 비교 |

### 8.2 성공 지표 (Success Metrics)

| 지표 | 목표값 | 측정 방법 |
|------|--------|----------|
| 일 활성 사용 (DAU) | 5회/일 이상 방문 | 세션 로그 |
| 태스크 완료율 | 업무시트 완료 상태 전환 70% 이상 | localStorage 상태 분포 |
| AI 사용 빈도 | 일 평균 3회 이상 | API 호출 로그 |
| 퀵런처 등록 수 | 평균 4개 이상 파일 등록 | localStorage 데이터 |
| 리텐션 | 1주 후 재방문율 60% | 쿠키 기반 방문 기록 |

### 8.3 사용자 검증 계획

**1단계 (현재): 내부 검증**
- 개발자 본인 2주 실사용
- 불편사항 데일리 노트에 기록

**2단계 (MVP 공개 후): 소수 사용자 인터뷰**
- EMR 기획자 5명 대상 1:1 인터뷰
- 핵심 질문: "기존 업무 방식과 비교해 어떤 점이 달랐나요?"

**3단계 (반복 개선)**
- 인터뷰 기반 기능 우선순위 재정렬
- A/B 테스트로 UI 개선 검증
