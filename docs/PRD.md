# MyDesk PRD (Product Requirements Document)

**버전**: 1.0.0
**작성일**: 2026-03-13
**작성자**: 기획자 (EMR 업계)
**상태**: MVP 개발 완료

---

## 1. 문제 정의 (Problem Statement)

### 1.1 배경

EMR(전자의무기록) 업계 기획자는 하루 평균 **7~10개 이상의 툴**을 넘나들며 업무를 수행한다.

| 도구 | 용도 |
|------|------|
| Figma | UI/UX 설계, 프로토타이핑 |
| Teams / Slack | 팀 커뮤니케이션 |
| Gmail / Outlook | 이메일 확인 |
| Notion / Confluence | 기획 문서 관리 |
| Excel / Google Sheets | 업무 추적 |
| Claude / ChatGPT / Gemini | AI 보조 |
| Google Calendar | 일정 관리 |

### 1.2 핵심 문제

> **"컨텍스트 스위칭 비용이 너무 크다"**

- 툴을 전환할 때마다 **집중 흐름(flow)이 끊김**
- 어떤 메시지를 읽었는지, 어떤 태스크가 남았는지 **파악에 시간 소요**
- 여러 AI 도구를 각각 열어야 해서 **AI 활용 마찰이 큼**
- 뽀모도로 같은 집중 관리 도구를 **별도로 실행해야 함**

### 1.3 영향

- 기획자 1인 기준, 하루 **40~60분이 툴 전환 및 컨텍스트 파악에 낭비**
- 중요 메시지·마감 놓침 → 품질 저하, 신뢰 손상
- AI를 활용하고 싶어도 진입 장벽으로 활용률 저조

## 2. 제품 목표 (Goals)

### 2.1 비전

> **"EMR 기획자를 위한 AI-Native 올인원 업무 허브"**
> 한 화면에서 모든 업무 맥락을 유지하며 일한다.

### 2.2 목표

| # | 목표 | 측정 방법 |
|---|------|----------|
| G1 | 툴 전환 횟수 50% 감소 | 일 평균 앱 전환 횟수 측정 |
| G2 | 업무 시작까지 걸리는 시간 단축 (< 30초) | 첫 액션까지 소요 시간 |
| G3 | AI 활용 빈도 증가 | 일 평균 AI 어시스턴트 사용 횟수 |
| G4 | 마감/메시지 누락 제로화 | 미읽은 알림 처리율 |

### 2.3 MVP 범위

이번 버전은 **혼자 쓰는 개인 대시보드 MVP**로, 실서버 연동보다 **동작하는 완성도**에 집중한다.

## 3. 사용자 정의 (Users)

### 주요 타겟

- **직군**: EMR 소프트웨어 회사 서비스 기획자
- **경력**: 2~8년차
- **특징**: 기술 이해도 높음, AI 툴 사용 경험 있음, 협업 툴 의존도 높음

### 유저 스토리

```
As a EMR 기획자,
I want to 하나의 화면에서 오늘의 태스크, AI 어시스턴트, 커뮤니케이션을 모두 확인하고 싶다
So that 툴 전환 없이 집중해서 일할 수 있다.
```

## 4. 기능 명세 (Feature Specification)

### 4.1 위젯 목록

#### REQ-01: 대시보드 레이아웃
- 2/3 + 1/3 그리드 레이아웃
- 위젯 표시/숨기기 토글
- 위젯 순서 변경 (위/아래)
- 레이아웃 프리셋 저장 (최대 3개)

#### REQ-02: 위젯 설정 모달
- 헤더 "설정" 버튼 → 모달 열기
- 각 위젯 표시/숨기기
- 위젯 순서 조정
- 프리셋: 기획 집중 모드 / 회의 많은 날 / 커스텀

#### REQ-03: 피그마 퀵런처
- Figma 파일 URL + 파일명 등록 (최대 8개)
- 색상 카드 썸네일 + 파일명 + 등록 시간 표시
- 카드 클릭 → 새 탭으로 Figma 오픈
- X 버튼 또는 우클릭 메뉴로 삭제
- localStorage 영속 저장

#### REQ-04: AI 어시스턴트 허브
- Groq(Llama 3.3), ChatGPT, Gemini 탭 전환
- Groq API 실시간 연동 (llama-3.3-70b-versatile)
- 대화 히스토리 스크롤
- ChatGPT / Gemini는 준비 중 표시
- 에러 시 친절한 메시지

#### REQ-05: 뽀모도로 타이머
- 25분 집중 / 5분 휴식 사이클
- 시작 / 일시정지 / 리셋
- 오늘 완료 세션 도트 표시 (최대 4개)
- 자동 전환

#### REQ-06: 업무 시트 뷰어
- Task / Status / Due / Priority 컬럼 테이블
- 상태 클릭 사이클: 대기 → 진행중 → 완료 → 보류
- 우선순위 색상 뱃지 (높음: 빨강, 보통: 노랑, 낮음: 회색)
- 태스크 추가 / 삭제
- localStorage 영속 저장

#### REQ-07: 데일리 노트
- 날짜별 체크리스트 + 자유 메모
- 완료 항목 아래로 정렬
- 오늘 날짜 자동 키 생성

#### REQ-08: 커뮤니케이션 피드
- 메일 + Teams 통합 피드 (목 데이터)
- 미읽음 카운트 뱃지
- 소스별 색상 구분 (메일: 파랑, Teams: 시안)

#### REQ-09: AI 조언 배너
- AI 조언 탭: 업무 컨텍스트 기반 조언 랜덤 로테이션
- 오늘의 운세 탭: Groq API로 생성, 하루 1회 캐싱
- 닫기 버튼

#### REQ-10: AI·기획 뉴스 피드
- Google News RSS 실시간 파싱 (서버사이드)
- AI기술 / UX기획 / 디자인툴 / 헬스IT 카테고리 자동 분류
- 30분 서버 캐시 + 클라이언트 캐시

#### REQ-11: Google 캘린더 뷰
- Google Calendar 공식 임베드 iframe
- 주간 / 월간 / 일정 뷰 전환
- 계정 이메일 입력으로 연결

#### REQ-12: 즐겨쓰는 앱 바로가기
- Figma, Claude, ChatGPT, Gemini, Teams, Excel, Mail
- 데스크탑 앱 프로토콜 우선 실행 (msteams://, figma:// 등)
- 앱 미설치 시 웹 fallback (1.5초 timeout)
- Teams, Mail 미읽음 뱃지 표시 (목 데이터)

## 8. 검증 계획 (Validation Plan)

### 8.1 핵심 가설

각 가설은 **검증 난이도**와 **측정 가능성**을 함께 명시한다.
MVP 단계에서는 정밀한 통계보다 방향성 확인을 목표로 한다.

| # | 가설 | 검증 기준 (성공) | 검증 기준 (실패) | 난이도 | 측정 도구 |
|---|------|----------------|----------------|--------|----------|
| H1 | MyDesk를 도입한 기획자는 기존 대비 외부 앱 전환 횟수가 줄어든다 | 자기 보고 기준 탭 전환 30% 이상 감소 | 변화 없거나 오히려 증가 | 높음 — 통제된 환경 필요 | 사용 전후 자기 관찰 일지 |
| H2 | 피그마 퀵런처는 Figma 파일 접근 경로를 단축한다 | 사용자 5명 중 4명이 "기존보다 빠르다" 응답 | 절반 이상이 "차이 없다" 응답 | 낮음 — 태스크 테스트로 쉽게 측정 | 사용성 테스트 관찰 + 시간 측정 |
| H3 | AI 어시스턴트를 대시보드 내에서 사용하면 AI 도구 전환 마찰이 줄어든다 | 인터뷰에서 "탭 안 열어도 된다"는 반응 3명 이상 | AI 어시스턴트 위젯 미사용 | 중간 — 정성적 검증 필요 | 인터뷰 + 행동 관찰 |
| H4 | 업무 시트를 통해 태스크 현황 파악 시간이 줄어든다 | 2주 실사용 후 "기존 방식보다 파악이 빠르다" 자기 평가 | "Notion/Excel이 더 편하다" 응답 | 중간 — 기존 도구와 비교 평가 필요 | 사용 전후 설문 |
| H5 | MyDesk를 브라우저 첫 탭으로 설정한 사용자는 2주 이상 지속 사용한다 | 5명 중 3명이 2주 후에도 메인 탭 유지 | 2주 내 대부분 기존 방식 복귀 | 낮음 — 장기 추적 가능 | 2주 후 재인터뷰 |

> **가설 우선순위**: H2, H5는 측정이 용이해 1차 검증에서 확인. H1, H3, H4는 2차 인터뷰에서 정성적으로 확인.

### 8.2 성공 지표 (Success Metrics) 및 측정 구현

#### 측정 방식: localStorage 이벤트 로그

외부 분석 서비스(GA, Mixpanel) 없이 **localStorage 기반 자체 이벤트 로그**로 측정한다.
MVP 규모(1인 개발, 소수 테스터)에는 충분하다.

**이벤트 로그 구조 (구현 예시):**
```typescript
// lib/analytics.ts
type EventName =
  | 'session_start'
  | 'figma_file_open'
  | 'ai_message_sent'
  | 'task_status_changed'
  | 'pomodoro_completed'
  | 'widget_settings_opened';

function logEvent(name: EventName, meta?: Record<string, string>) {
  const key = 'mydesk:events';
  const logs = JSON.parse(localStorage.getItem(key) ?? '[]');
  logs.push({ name, ts: Date.now(), date: new Date().toISOString().slice(0, 10), ...meta });
  // 최대 500건 유지
  if (logs.length > 500) logs.splice(0, logs.length - 500);
  localStorage.setItem(key, JSON.stringify(logs));
}
```

**데이터 추출 방법 (브라우저 콘솔에서 실행):**
```javascript
JSON.parse(localStorage.getItem('mydesk:events') ?? '[]')
```

#### 지표별 측정 세부 계획

| 지표 | 목표값 | 측정 이벤트 | 계산 방식 | 수집 기간 |
|------|--------|------------|----------|----------|
| 일 방문 빈도 | 3회/일 이상 (보수적) | `session_start` | 날짜별 이벤트 그룹 카운트 | 2주 |
| 피그마 퀵런처 사용률 | 등록 파일 1개 이상 + 주 3회 이상 클릭 | `figma_file_open` | 주간 이벤트 수 | 2주 |
| AI 어시스턴트 사용 빈도 | 일 평균 2회 이상 | `ai_message_sent` | 날짜별 카운트 평균 | 2주 |
| 태스크 완료율 | 등록 태스크 중 완료 전환 50% 이상 | localStorage 직접 조회 | `done` 상태 비율 | 2주 후 스냅샷 |
| 2주 리텐션 | 5명 중 3명 지속 사용 | 2주 후 재인터뷰 확인 | 정성 응답 | 2주차 |

> **목표값 조정 이유**: 초기 PRD의 "DAU 5회", "완료율 70%"는 근거 없이 낙관적이었다.
> 실제 개인 업무 도구의 초기 정착률을 감안해 보수적으로 하향 조정.

### 8.3 사용자 검증 계획

#### 1단계 — 내부 검증 (출시 후 1~2주)

**대상**: 기획자 본인 1인 자기 관찰

**측정 항목:**
- 매일 업무 종료 시 데일리 노트에 다음 3가지 기록
  1. MyDesk를 몇 번 열었나?
  2. 어떤 위젯을 가장 많이 썼나?
  3. 불편하거나 안 쓴 기능은?

**기간**: 2주 (10 영업일)

**판단 기준**: 2주 후에도 매일 사용하고 있으면 → 2단계 진행
반대로 1주 안에 포기하면 → UX 개선 먼저

---

#### 2단계 — 소수 사용자 테스트 (출시 후 3~4주)

**대상**: EMR 업계 기획자 5명
- 모집 기준: 경력 2~8년차, Figma + AI 도구 사용 경험 있는 기획자
- 모집 방법: 사내 동료 + LinkedIn 네트워크

**진행 방식: 태스크 기반 사용성 테스트 (30분)**

| 순서 | 태스크 | 관찰 포인트 |
|------|--------|------------|
| T1 | 피그마 파일 2개 등록하고 열어보기 | 모달 UX, URL 입력 흐름 |
| T2 | AI 어시스턴트에서 기획 질문 1개 입력하기 | 탭 인지, 입력 흐름 |
| T3 | 오늘 할 태스크 3개 업무 시트에 추가하기 | 컬럼 인지, 상태 변경 |
| T4 | 뽀모도로 타이머 1세션 시작하기 | 버튼 직관성 |
| T5 | 위젯 하나 숨기고 다시 켜기 | 설정 모달 접근성 |

**사후 인터뷰 질문 (5~10분):**
1. 오늘 사용해본 기능 중 실제 업무에서 쓸 것 같은 게 있나요? 이유는?
2. 반대로 필요 없을 것 같은 기능이 있었나요?
3. 기존에 Figma 파일을 어떻게 접근하셨나요? 오늘 방식과 비교하면?
4. 이 대시보드를 브라우저 첫 탭으로 설정할 의향이 있나요? (1~5점)
5. 동료 기획자에게 추천하시겠어요? (NPS 0~10점)

**분석 방법:**
- 태스크 완료율 + 소요 시간 기록
- 인터뷰 내용 어피니티 다이어그램으로 클러스터링
- NPS 기록 (10점 만점, 9~10: 추천, 7~8: 중립, 0~6: 비추천)

#### 3단계 — 반복 개선 (테스트 이후)

**우선순위 재설정 기준:**
- 5명 중 3명 이상이 언급한 불편 사항 → 즉시 개선 대상
- 1~2명만 언급 → 백로그 추가 후 재검토
- 아무도 안 쓴 위젯 → 기본 숨김 처리 고려

**A/B 테스트 후보 (우선순위순):**
1. 피그마 퀵런처 레이아웃 — 그리드 4열 vs 3열
2. AI 조언 배너 — 항상 표시 vs 접을 수 있는 토글
3. 업무 시트 — 테이블 형태 vs 칸반 보드 형태
