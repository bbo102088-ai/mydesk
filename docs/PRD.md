## 1. 문제 정의 (Problem Statement)

### 1.1 배경

EMR(전자의무기록) 업계 기획자는 하루 평균 **7~10개 이상의 툴**을 넘나들며 업무를 수행한다.

| 도구 | 용도 |
|------|------|
| Figma | UI/UX 설계, 프로토타이핑 |
| Teams / Slack | 팀 커뮤니케이션 |
| Gmail / Outlook | 이메일 확인 |
| Notion / Confluence | 기획 문서 관리 |
| Excel / Google Sheets | 업무 추적 |
| Claude / ChatGPT / Gemini | AI 보조 |
| Google Calendar | 일정 관리 |

### 1.2 핵심 문제

> **"컨텍스트 스위칭 비용이 너무 크다"**

- 툴을 전환할 때마다 **집중 흐름(flow)이 끊김**
- 어떤 메시지를 읽었는지, 어떤 태스크가 남았는지 **파악에 시간 소요**
- 여러 AI 도구를 각각 열어야 해서 **AI 활용 마찰이 큼**
- 뽀모도로 같은 집중 관리 도구를 **별도로 실행해야 함**

### 1.3 영향

- 기획자 1인 기준, 하루 **40~60분이 툴 전환 및 컨텍스트 파악에 낭비**
- 중요 메시지·마감 놓침 → 품질 저하, 신뢰 손상
- AI를 활용하고 싶어도 진입 장벽으로 활용률 저조

---

### 1.4 시장 규모 (Market Size)

MyDesk는 **EMR 업계 기획자**를 1차 타겟으로 하며, 중장기적으로 의료 IT 업계 전반의 지식 노동자로 확장한다.

#### TAM — 전체 잠재 시장

| 시장 | 규모 | 출처/근거 |
|------|------|----------|
| 글로벌 EMR 시장 | **USD 38.5B (2024), 연 CAGR 8.6%** | Grand View Research 2024 |
| 국내 EMR·의료정보 SW 시장 | **약 6,200억 원 (2024)** | 한국보건산업진흥원 |
| 글로벌 업무 생산성 도구 시장 | **USD 57.3B (2023), 연 CAGR 13.4%** | MarketsandMarkets |

#### SAM — 유효 접근 시장

국내 EMR·의료정보 소프트웨어 기업 기준:

- 주요 기업: 유비케어, 이지케어텍, 비트컴퓨터, 인성정보, 솔루션링크 등 **50~80개 업체**
- 기업당 서비스 기획자·PM 평균 **5~20명** 추정
- **국내 타겟 인력 규모: 약 500~1,500명**

#### SOM — 실질 확보 가능 시장 (MVP 단계)

- 1차 목표: 본인 포함 **사내 기획팀 10~20명** 자체 검증
- 2차 목표: 커뮤니티(기획자 LinkedIn, OKKY, 플래텀) 통해 **50~200명** 베타 사용자 확보
- MyDesk가 EMR 이외 의료 IT(HIS, PACS, 원격의료 등)로 확장 시 SAM **3배 이상** 확대 가능

> **MVP 단계 전략**: 시장 규모보다 **"기획자가 매일 열고 싶은 첫 탭"**이 되는 것이 핵심.
> 소수 강성 사용자(Power User) 확보 → 입소문 → 유기적 확산.

#### 핵심 차별화 요소 (Unique Value Proposition)

1. **EMR 도메인 특화 AI**: 업무 시트의 태스크가 AI 시스템 프롬프트에 자동 주입 → 범용 AI 대비 즉시 실용적인 답변
2. **제로 셋업**: 설치 없이 브라우저 URL 하나로 시작, localStorage만으로 동작
3. **피그마 퀵런처**: EMR 기획자의 핵심 도구인 Figma를 1클릭으로 연결하는 기능은 타 대시보드에 없음
4. **컨텍스트 스위칭 제로**: 뉴스·캘린더·AI·타이머·태스크를 한 탭에서 해결

## 2. 제품 목표 (Goals)

### 2.1 비전

> **"EMR 기획자를 위한 AI-Native 올인원 업무 허브"**
> 한 화면에서 모든 업무 맥락을 유지하며 일한다.

### 2.2 목표

| # | 목표 | 측정 방법 |
|---|------|----------|
| G1 | 툴 전환 횟수 50% 감소 | 일 평균 앱 전환 횟수 측정 |
| G2 | 업무 시작까지 걸리는 시간 단축 (< 30초) | 첫 액션까지 소요 시간 |
| G3 | AI 활용 빈도 증가 | 일 평균 AI 어시스턴트 사용 횟수 |
| G4 | 마감/메시지 누락 제로화 | 미읽은 알림 처리율 |

### 2.3 MVP 범위

이번 버전은 **혼자 쓰는 개인 대시보드 MVP**로, 실서버 연동보다 **동작하는 완성도**에 집중한다.

- **직군**: EMR 소프트웨어 회사 서비스 기획자
- **경력**: 2~8년차

```
As a EMR 기획자,
I want to 하나의 화면에서 오늘의 태스크, AI 어시스턴트, 커뮤니케이션을 모두 확인하고 싶다
So that 툴 전환 없이 집중해서 일할 수 있다.
```

## 4. 기능 명세 (Feature Specification)

### 4.1 위젯 목록

#### REQ-01: 대시보드 레이아웃
- 2/3 + 1/3 그리드 레이아웃
- 위젯 표시/숨기기 토글
- 위젯 순서 변경 (위/아래)
- 레이아웃 프리셋 저장 (최대 3개)

#### REQ-02: 위젯 설정 모달
- 헤더 "설정" 버튼 → 모달 열기
- 각 위젯 표시/숨기기
- 위젯 순서 조정
- 프리셋: 기획 집중 모드 / 회의 많은 날 / 커스텀

#### REQ-03: 피그마 퀵런처
- Figma 파일 URL + 파일명 등록 (최대 8개)
- 색상 카드 썸네일 + 파일명 + 등록 시간 표시
- 카드 클릭 → 새 탭으로 Figma 오픈
- X 버튼 또는 우클릭 메뉴로 삭제
- localStorage 영속 저장

#### REQ-04: AI 어시스턴트 허브
- Groq(Llama 3.3), ChatGPT, Gemini 탭 전환
- Groq API 실시간 연동 (llama-3.3-70b-versatile)
- 대화 히스토리 스크롤
- ChatGPT / Gemini는 준비 중 표시
- 에러 시 친절한 메시지

#### REQ-05: 뽀모도로 타이머
- 25분 집중 / 5분 휴식 사이클
- 시작 / 일시정지 / 리셋
- 오늘 완료 세션 도트 표시 (최대 4개)
- 자동 전환

#### REQ-06: 업무 시트 뷰어
- Task / Status / Due / Priority 컬럼 테이블
- 상태 클릭 사이클: 대기 → 진행중 → 완료 → 보류
- 우선순위 색상 뱃지 (높음: 빨강, 보통: 노랑, 낮음: 회색)
- 태스크 추가 / 삭제
- localStorage 영속 저장

#### REQ-07: 데일리 노트
- 날짜별 체크리스트 + 자유 메모
- 완료 항목 아래로 정렬
- 오늘 날짜 자동 키 생성

#### REQ-08: 커뮤니케이션 피드
- 메일 + Teams 통합 피드 (목 데이터)
- 미읽음 카운트 뱃지
- 소스별 색상 구분 (메일: 파랑, Teams: 시안)

#### REQ-09: AI 조언 배너
- AI 조언 탭: 업무 컨텍스트 기반 조언 랜덤 로테이션
- 오늘의 운세 탭: Groq API로 생성, 하루 1회 캐싱
- 닫기 버튼

#### REQ-10: AI·기획 뉴스 피드
- Google News RSS 실시간 파싱 (서버사이드)
- AI기술 / UX기획 / 디자인툴 / 헬스IT 카테고리 자동 분류
- 30분 서버 캐시 + 클라이언트 캐시

#### REQ-11: Google 캘린더 뷰
- Google Calendar 공식 임베드 iframe
- 주간 / 월간 / 일정 뷰 전환
- 계정 이메일 입력으로 연결

#### REQ-12: 즐겨쓰는 앱 바로가기
- Figma, Claude, ChatGPT, Gemini, Teams, Excel, Mail

## 8. 검증 계획 (Validation Plan)

### 8.1 핵심 가설

각 가설은 **검증 난이도**와 **측정 가능성**을 함께 명시한다.
MVP 단계에서는 정밀한 통계보다 방향성 확인을 목표로 한다.

#### 가설 인과 구조 (Causal Chain)

MyDesk의 가치 사슬은 아래와 같은 인과 순서로 성립한다.
**기능 가설(H2·H3·H4)이 각각 검증되어야 H1(컨텍스트 스위칭 감소)이 성립하고,
H1이 성립해야 비로소 H5(장기 리텐션)가 기대 가능하다.**

```
H2 (피그마 접근 단축)  ─┐
H3 (AI 전환 마찰 감소) ─┤→ H1 (전체 툴 전환 30% 감소) → H5 (2주 리텐션)
H4 (태스크 파악 가속)  ─┘
```

> **논리**: H1은 결과 가설(outcome)이고 H2~H4는 원인 가설(driver)이다.
> H2~H4 중 하나라도 실패하면 H1 달성은 어렵다.
> H1 없이는 H5(습관 형성)가 발생할 동기가 부족하다.

#### 우선순위 논리

| 우선순위 | 가설 | 이유 |
|---------|------|------|
| 1차 검증 | H2 | 측정 용이(태스크 테스트), 피그마가 EMR 기획자의 핵심 툴 |
| 1차 검증 | H3 | analytics.ts `ai_message_sent` 이벤트로 정량 측정 가능 |
| 2차 검증 | H4 | 2주 실사용 데이터 필요, `task_status_changed` 이벤트 누적 후 판단 |
| 2차 검증 | H1 | H2·H3·H4 검증 후 종합 판단, 자기 관찰 일지 병행 |
| 3차 검증 | H5 | 2주 이상 지속 사용 데이터 수집 후 판단 (리텐션 스트릭) |

> H2·H3을 먼저 검증하는 이유: 빠르게 측정 가능하고, 두 가설이 실패하면
> H1·H5로 가는 인과 고리가 끊기므로 이른 피벗 판단이 가능하다.

#### 가설별 상세

| # | 가설 | 검증 기준 (성공) | 검증 기준 (실패) | 난이도 | 측정 도구 |
|---|------|----------------|----------------|--------|----------|
| H2 | 피그마 퀵런처는 Figma 파일 접근 경로를 단축한다 | 사용자 5명 중 4명이 "기존보다 빠르다" 응답 | 절반 이상이 "차이 없다" 응답 | 낮음 — 태스크 테스트로 쉽게 측정 | 사용성 테스트 + `figma_file_open` 이벤트 |
| H3 | AI 어시스턴트를 대시보드 내에서 사용하면 AI 도구 전환 마찰이 줄어든다 | 일 평균 `ai_message_sent` 2회 이상 & "탭 안 열어도 된다" 반응 3명 이상 | AI 어시스턴트 위젯 `funnelConversion` 10% 미만 | 중간 — 정성+정량 병행 | 인터뷰 + analytics funnel |
| H4 | 업무 시트를 통해 태스크 현황 파악 시간이 줄어든다 | 2주 실사용 후 `task_status_changed` 이벤트 주 5회 이상 & "기존 방식보다 빠르다" 자기 평가 | "Notion/Excel이 더 편하다" 응답 또는 이벤트 주 2회 미만 | 중간 — 기존 도구와 비교 평가 필요 | 이벤트 로그 + 사용 전후 설문 |
| H1 | MyDesk를 도입한 기획자는 기존 대비 외부 앱 전환 횟수가 줄어든다 | 자기 보고 기준 탭 전환 30% 이상 감소 & H2·H3·H4 중 2개 이상 성공 | 변화 없거나 오히려 증가 | 높음 — H2·H3·H4 선행 필요 | 자기 관찰 일지 + 이벤트 로그 종합 |
| H5 | MyDesk를 브라우저 첫 탭으로 설정한 사용자는 2주 이상 지속 사용한다 | `retentionStreak()` 기준 10일 이상 방문 (5명 중 3명) | 2주 내 대부분 기존 방식 복귀 (`retentionStreak` < 5일) | 낮음 — 장기 추적 가능 | analytics `retentionStreak` + 2주 후 재인터뷰 |

### 8.2 성공 지표 (Success Metrics) 및 측정 구현

#### 측정 방식: localStorage 이벤트 로그

외부 분석 서비스(GA, Mixpanel) 없이 **localStorage 기반 자체 이벤트 로그**로 측정한다.
MVP 규모(1인 개발, 소수 테스터)에는 충분하다.

**실제 구현된 이벤트 목록 (`src/lib/analytics.ts`):**

| 이벤트 | 발생 시점 | 연결 가설 |
|--------|----------|----------|
| `session_start` | 앱 진입 시 (SessionTracker) | H1, H5 |
| `figma_file_open` | 피그마 파일 카드 클릭 시 | H2 |
| `ai_message_sent` | AI 직접 입력 전송 시 | H3 |
| `ai_template_used` | 원클릭 템플릿 사용 시 | H3 |
| `task_status_changed` | 태스크 상태 버튼 클릭 시 | H4 |
| `task_added` | 새 태스크 추가 시 | H4 |
| `pomodoro_session_completed` | 25분 집중 완료 시 | H1 |
| `quick_app_launched` | 퀵앱 바 버튼 클릭 시 | H1 |

**데이터 추출 방법 (브라우저 콘솔에서 실행):**
```javascript
// 원시 이벤트 전체 조회
JSON.parse(localStorage.getItem('mydesk:events') ?? '[]')

// 날짜별 이벤트 집계 요약
(await import('/src/lib/analytics.ts')).summarizeEvents()

// 특정 이벤트 일별 카운트
(await import('/src/lib/analytics.ts')).dailyCount('ai_message_sent')

// 연속 방문 일수 (리텐션 스트릭)
(await import('/src/lib/analytics.ts')).retentionStreak()

// 기능 전환율 퍼널 (세션 대비 각 기능 사용률)
(await import('/src/lib/analytics.ts')).funnelConversion()
```

---

#### 데이터 분석 방법론

**① 리텐션 분석 (Retention Streak)**
- `retentionStreak()` 함수: `session_start` 이벤트 기준 현재 기준 며칠 연속 방문했는지 계산
- 판단 기준: 10일 이상 = H5 성공 / 5일 미만 = 조기 이탈 위험

**② 퍼널 분석 (Funnel Conversion)**
- `funnelConversion()` 함수: 세션 방문자 중 각 핵심 기능을 실제 사용한 비율 계산
- 퍼널 단계: `session_start` → `figma_file_open` / `ai_message_sent` / `task_status_changed`
- 기능별 전환율이 20% 미만이면 해당 위젯의 UX 개선 우선순위 상향

**③ 기능 채택률 분석 (Feature Adoption Rate)**
- 전체 이벤트 중 각 이벤트 유형의 비율 측정
- 특정 기능이 5% 미만이면 → 접근성 문제 또는 필요성 부족으로 판단

**④ 행동 패턴 분석 (Behavioral Pattern)**
- `summarizeEvents()` 결과를 날짜별로 시각화해 요일별 사용 패턴 확인
- 주 5일 이상 사용 여부가 H1 검증의 보조 지표

---

#### 지표별 측정 세부 계획

| 지표 | 목표값 | 측정 방법 | 분석 함수 | 수집 기간 |
|------|--------|----------|----------|----------|
| 일 방문 빈도 | 3회/일 이상 | `session_start` 날짜별 카운트 | `dailyCount('session_start')` | 2주 |
| 피그마 퀵런처 사용률 | 주 3회 이상 클릭 | `figma_file_open` 주간 합산 | `dailyCount('figma_file_open')` | 2주 |
| AI 어시스턴트 사용 빈도 | 일 평균 2회 이상 | `ai_message_sent` + `ai_template_used` | `summarizeEvents()` | 2주 |
| 태스크 관리 활성도 | 주 5회 이상 상태 변경 | `task_status_changed` 주간 합산 | `dailyCount('task_status_changed')` | 2주 |
| 기능 전환율 (퍼널) | 세션 대비 각 기능 20% 이상 | 세션 수 대비 기능 이벤트 비율 | `funnelConversion()` | 2주 |
| 연속 방문 (리텐션) | 10일 이상 연속 | 연속 방문 일수 | `retentionStreak()` | 2주 |
| 2주 리텐션 (정성) | 5명 중 3명 지속 사용 | 2주 후 재인터뷰 확인 | 정성 응답 | 2주차 |

> **목표값 근거**: 개인 업무 도구 초기 정착률 벤치마크(Notion 1개월 리텐션 ~40%, Todoist DAU 패턴)를
> 참고해 보수적으로 설정. 초기 5인 테스트 단계에서는 방향성 확인이 목적이며,
> 통계적 유의성보다 정성 인터뷰와 병행해 해석한다.

### 8.3 사용자 검증 계획

#### 1단계 — 내부 검증 (출시 후 1~2주)

**대상**: 기획자 본인 1인 자기 관찰

**측정 항목:**
- 매일 업무 종료 시 데일리 노트에 다음 3가지 기록
  1. MyDesk를 몇 번 열었나?
  2. 어떤 위젯을 가장 많이 썼나?
  3. 불편하거나 안 쓴 기능은?

**기간**: 2주 (10 영업일)

**판단 기준**: 2주 후에도 매일 사용하고 있으면 → 2단계 진행
반대로 1주 안에 포기하면 → UX 개선 먼저

---

#### 2단계 — 소수 사용자 테스트 (출시 후 3~4주)

**대상**: EMR 업계 기획자 5명
- 모집 기준: 경력 2~8년차, Figma + AI 도구 사용 경험 있는 기획자
- 모집 방법: 사내 동료 + LinkedIn 네트워크

**진행 방식: 태스크 기반 사용성 테스트 (30분)**

| 순서 | 태스크 | 관찰 포인트 |
|------|--------|------------|
| T1 | 피그마 파일 2개 등록하고 열어보기 | 모달 UX, URL 입력 흐름 |
| T2 | AI 어시스턴트에서 기획 질문 1개 입력하기 | 탭 인지, 입력 흐름 |
| T3 | 오늘 할 태스크 3개 업무 시트에 추가하기 | 컬럼 인지, 상태 변경 |
| T4 | 뽀모도로 타이머 1세션 시작하기 | 버튼 직관성 |
| T5 | 위젯 하나 숨기고 다시 켜기 | 설정 모달 접근성 |

**사후 인터뷰 질문 (5~10분):**
1. 오늘 사용해본 기능 중 실제 업무에서 쓸 것 같은 게 있나요? 이유는?
2. 반대로 필요 없을 것 같은 기능이 있었나요?
3. 기존에 Figma 파일을 어떻게 접근하셨나요? 오늘 방식과 비교하면?
4. 이 대시보드를 브라우저 첫 탭으로 설정할 의향이 있나요? (1~5점)
5. 동료 기획자에게 추천하시겠어요? (NPS 0~10점)

**분석 방법:**
- 태스크 완료율 + 소요 시간 기록
- 인터뷰 내용 어피니티 다이어그램으로 클러스터링
- NPS 기록 (10점 만점, 9~10: 추천, 7~8: 중립, 0~6: 비추천)

---

#### 3단계 — 반복 개선 (테스트 이후)

**우선순위 재설정 기준:**
- 5명 중 3명 이상이 언급한 불편 사항 → 즉시 개선 대상
- 1~2명만 언급 → 백로그 추가 후 재검토
- 아무도 안 쓴 위젯 → 기본 숨김 처리 고려

**A/B 테스트 후보 (우선순위순):**
1. 피그마 퀵런처 레이아웃 — 그리드 4열 vs 3열
2. AI 조언 배너 — 항상 표시 vs 접을 수 있는 토글
3. 업무 시트 — 테이블 형태 vs 칸반 보드 형태
